## 共性问题

1. **TEST.md 模板填充率低**: 两个 change 的 TEST.md 均有大量占位符，代码路径覆盖图为空模板
2. **Quality Gate checkbox**: 质量门禁复选框普遍未勾选，形式化大于实质性
3. **5-Round Coverage 标注不精确**: "待定" 和 "跳过" 语义混淆
4. **UAT 脚本缺失**: 两个 change 都未提供手动验证脚本
5. **占位符残留**: 各阶段模板渲染后常有 `[流程名]` / `[描述]` / `[待定]` 等占位
6. **Gate 规则漂移**: 模板 headings（如 `## Step 1: User Stories`）与新 gate 期望（`## User Stories`）不一致

---

## 改进建议

1. 模板 hbs 中移除最终占位符，或渲染时检测并警告未填充项
2. TEST.md 的代码路径覆盖图在无实现时允许 `N/A` 声明
3. Gate 检查逻辑对齐模板实际输出格式（heading naming）
4. 增加 `--skip-quality-gate` 开关用于已知轻量变更
5. ACL 级别的 NFR 校验应考虑纯审计/CLI 变更的宽松路径
